# kernelcode_generate 开发检查清单

## 需求确认

- 明确用户可见结果和目标后端。
- 判断变更是增量兼容还是破坏性修改。
- 枚举影响层：DSL、中间表示（IR）、编译器（compiler）、后端（backend）、CLI、测试（tests）、文档（docs）。

## 实现门禁

- 新 API 在可行情况下保持向后兼容。
- 中间表示（IR）节点命名保持一致且语义明确。
- 后端发射器（emitter）输出保持确定性。
- 为新路径补充或更新示例代码。

## 测试门禁

- 先运行定向测试。
- 条件允许时运行全量测试。
- 对每个受影响后端断言关键生成代码片段。
- 新行为至少增加一个边界/错误路径测试。

## 交付门禁

- 外部行为变化时更新 README 或架构说明。
- 记录执行命令及结果。
- 记录已知限制和后续事项。
