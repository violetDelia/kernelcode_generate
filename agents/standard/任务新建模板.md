# 任务新建模板

## 文件说明
- 功能说明：统一任务描述格式，让分发对象能直接判断目标、计划书和记录文件。
- 使用示例：创建任务时，把本文件的短格式放入 `-new -info "<...>"`。

## 通用格式
- 有计划书：`<topic>-plan-execute；计划书：《<plan path>》；任务目标：<可执行动作 + 对象 + 验收结果>；任务链记录：<record path>`
- 无计划书：`<type>；任务目标：<可执行动作 + 对象 + 验收结果>；任务链记录：<record path>`
- 计划修复：`<topic>-plan-execute；计划书：《<plan path>》；任务目标：修复：<最小阻断项 + 验收结果>；任务链记录：<record path>`

## 写法要求
- `任务目标` 必须可直接执行，写清要改什么、增加什么能力、跑通什么测试或合同入口。
- 有计划书时，只创建计划级任务；计划内小任务卡保留在计划正文中。
- `任务链记录` 写当前链路沿用的记录文件。
- `worktree`、验收细节、长背景和长期规划放在任务分发消息、计划书或记录文件中，不塞进短描述。
- 公开 `API` 变更任务必须在计划书、专题 `spec` 或记录中写明用户确认来源。

## 合格示例
- `kernel-split-plan-execute；计划书：《ARCHITECTURE/plan/kernel_split_pass_green_plan.md》；任务目标：按计划书完成 tuner.param 动态 tile 切分公开接口、实现、测试与验收资产闭环；任务链记录：agents/codex-multi-agents/log/task_records/2026/15/20260405-kernel-split.md`
- `execute；任务目标：为 npu_demo DMA slice 增加 include 承接，并跑通 pytest -q test/include/npu_demo；任务链记录：agents/codex-multi-agents/log/task_records/2026/15/20260405-npu-demo.md`
- `default-lowering-plan-execute；计划书：《ARCHITECTURE/plan/pass_pipeline_registration_refactor_green_plan.md》；任务目标：修复：补齐 default-lowering pass 注册并跑通 pytest -q test/passes/test_pipeline_default_lowering.py；任务链记录：agents/codex-multi-agents/log/task_records/2026/15/20260413-default-lowering.md`

## 不合格写法
- `继续推进`：目标不可执行。
- `整理一下结构`：没有对象和验收结果。
- `顺手把后面都做完`：超出当前任务边界。
- `S2-实现`：计划内小任务卡不是独立任务。
