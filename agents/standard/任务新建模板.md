# 任务新建模板

## 文件说明
- 功能说明：统一 `codex-multi-agents-task.sh -new -info` 的任务描述格式，减少不同角色之间的沟通歧义。
- 使用示例：`demo-S2-审查；计划书：《ARCHITECTURE/plan/demo.md》；任务目标：确认 spec / 实现 / 测试一致；任务链记录：20260405-demo.md`


## 通用规则
- 任务描述默认使用单行短格式，便于直接放入 `-new -info "<...>"`。
- 有计划书时，默认写成 `"<plan>-<S#>-<阶段>；计划书：《<计划书路径>》；任务目标：<...>；任务链记录：<...>"`。
- 无计划书时，不写 `计划书` 字段，默认写成 `"<阶段>；任务目标：<...>；任务链记录：<...>"`。
- `阶段` 只使用：`spec`、`实现`、`测试`、`重构`、`审查`、`复审`、`合并`。
- `任务目标` 只写当前阶段直接产出，不写泛化目标或长期规划。
- `任务链记录` 一律写当前链路沿用的记录文件短名。
- `<plan>` 只写计划简称或计划文件核心名，不重复整条路径；完整路径放在 `计划书：《...》` 中。
- 默认不把 `worktree`、`gate`、`验收标准` 塞进任务描述；只有管理员明确要求时才追加。

## 推荐模板
有计划书：`<plan>-<S#>-<阶段>；计划书：《<计划书路径>》；任务目标：<当前阶段直接产出>；任务链记录：<记录文件短名>`

无计划书：`<阶段>；任务目标：<当前阶段直接产出>；任务链记录：<记录文件短名>`

## 简短示例
- 有计划书：`kernel-split-S1-spec；计划书：《ARCHITECTURE/plan/kernel_split_pass_green_plan.md》；任务目标：冻结 tuner.param 动态 tile 切分合同；任务链记录：20260405-kernel-split.md`
- 无计划书：`实现；任务目标：补齐 npu_demo 的 DMA slice include 承接；任务链记录：20260405-npu-demo.md`
