# 任务新建模板

## 文件说明
- 功能说明：统一 `codex-multi-agents-task.sh -new -info` 的任务描述格式，减少不同角色之间的沟通歧义。
- 使用示例：`demo-S2-审查；计划书：《ARCHITECTURE/plan/demo.md》；任务目标：确认 spec / 实现 / 测试一致；任务链记录：20260405-demo.md`


## 通用规则
- 任务描述默认使用单行短格式，便于直接放入 `-new -info "<...>"`。
- 有计划书时，默认写成 `"<plan>-<S#>-<阶段>；计划书：《<计划书路径>》；任务目标：<...>；任务链记录：<...>"`。
- 无计划书时，不写 `计划书` 字段，默认写成 `"<阶段>；任务目标：<...>；任务链记录：<...>"`。
- `阶段` 只使用：`spec`、`实现`、`测试`、`重构`、`审查`、`复审`、`合并`。
- `任务目标` 只写当前阶段直接产出，不写泛化目标或长期规划。
- `任务链记录` 一律写当前链路沿用的记录文件短名。
- `<plan>` 只写计划简称或计划文件核心名，不重复整条路径；完整路径放在 `计划书：《...》` 中。
- 默认不把 `worktree`、`gate`、`验收标准` 塞进任务描述；只有管理员明确要求时才追加。
- 若计划书终验不通过，需要补建修复任务时，仍沿用原计划书路径；`任务目标` 第一短句直接写 `修复：<最小阻断项>`，避免和正常阶段混淆。

## 推荐模板
有计划书：`<plan>-<S#>-<阶段>；计划书：《<计划书路径>》；任务目标：<当前阶段直接产出>；任务链记录：<记录文件短名>`

无计划书：`<阶段>；任务目标：<当前阶段直接产出>；任务链记录：<记录文件短名>`

计划修复任务：`<plan>-<S#>-<阶段>；计划书：《<计划书路径>》；任务目标：修复：<最小阻断项>；任务链记录：<记录文件短名>`

## 失败示例

| 失败示例 | 为什么不合格 | 应如何改 |
| --- | --- | --- |
| `demo-S2-实现；计划书：《ARCHITECTURE/plan/demo.md》；任务目标：顺手把后面所有阶段一起做完；任务链记录：20260415-demo.md` | `任务目标` 扩成长期规划，审查人无法判断本阶段是否完成 | 只写当前阶段直接产出，例如“补齐 xx 示例并完成文本核对” |
| `demo-S2-实现；计划书：《ARCHITECTURE/plan/demo.md》；任务目标：补齐示例；任务链记录：20260415-demo.md；worktree=wt-demo；验收标准=全部通过` | 把 `worktree`、验收说明塞进任务描述，和标准短格式混写 | `worktree` 放在任务分发消息里，验收标准放计划书或记录文件里 |
| `实现；任务目标：补齐示例` | 缺少 `任务链记录`，后续无法确定沿用哪份日志 | 补成 `实现；任务目标：补齐示例；任务链记录：20260415-demo.md` |
| `demo-S2-实现；任务目标：补齐示例；任务链记录：20260415-demo.md` | 明明有计划书却漏写 `计划书：《...》`，会导致执行边界不清 | 有计划书时必须显式写出完整计划书路径 |
