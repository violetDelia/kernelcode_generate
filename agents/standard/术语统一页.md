# 术语统一页

## 文件说明
- 功能说明：统一多角色协作中常用术语，避免同词不同义。
- 使用示例：计划书、任务记录或审查意见出现歧义时，按本文件解释。

## 术语
| 术语 | 含义 |
| --- | --- |
| 用户 | 需求与公开 API 取舍的最终确认者 |
| 架构师 | 计划、专题 `spec`、架构裁定、互评和终验负责人 |
| 管理员 | 任务状态、分发、续接和归档协调人 |
| execute | 计划内规格、实现、测试与验收闭环任务 |
| review | 对 execute 产物做审查或复审 |
| merge | 合入已通过链路的允许文件 |
| 计划级任务 | 一个计划书对应的唯一 `execute` 大任务 |
| 计划内小任务卡 | 计划书内部的 `S1/S2/...` 可执行任务块，不是独立任务状态 |
| 计划级流转 | `execute -> review -> 架构复核 / 终验 -> merge/归档`；失败统一回 `execute` |
| 公开 API | `spec/API 列表`、函数 / 类 / 方法签名、工具入口、脚本参数、include 接口和稳定错误语义 |
| 非公开 API | 未进入 `spec` 和文件级 `API 列表` 的 helper、内部函数、隐藏方法或临时兼容入口 |
| 合同真源 | 决定公开行为的优先依据，例如 `spec`、pytest、`expectation` 或脚本 |
| expectation | `expectation/` 下的合同资产，只允许用户或架构师在授权范围内修改；只有被当前计划列为必过合同验收资产时才是当前阻断项 |
| Diff 反推测试 | 根据实际改动文件推导出的 pytest、测试脚本或可运行本地脚本 |
| 合同验收 | 对 `expectation` 或计划指定验收资产的执行与核对 |
| 最新同步现场 | 已同步到当前目标主线或指定远端基线的执行目录 |
| 阻断项 | 不修复就不能通过当前阶段的最小问题 |

## 判定
- 术语解释与个人提示词冲突时，以标准文档和 `AGENTS.md` 为准。
- 标准文档之间冲突时，先暂停并请求架构师裁定。
