# 术语统一页

## 文件说明
- 功能说明：统一 `agents/standard` 中的高频术语定义，降低跨文档阅读时的歧义。
- 使用示例：当文档提到 `worktree`、`主仓`、`阻塞记录` 时，先回到本页确认这些词在当前仓库中的固定含义。
- 使用示例：当管理员或执行人需要区分“任务”和“阶段”时，先看本页的术语表，再回到对应标准文档。

## 术语表

| 术语 | 定义 | 不等于什么 | 首次出现文档 |
| --- | --- | --- | --- |
| 任务 | `TODO.md` 中可被分发、续接、暂停、完成的一条执行单元，带有任务号、类型、`worktree` 与记录文件 | 计划书中的抽象目标，或口头待办 | [`协作执行通用规则.md`](../../agents/standard/协作执行通用规则.md) |
| 计划内小任务卡 | 计划书中按 `S1/S2/...` 拆分的可执行说明块，必须写清目标、边界、执行步骤、验收与记录要求 | 具体任务号，或独立 `TODO` 状态 | [`计划书标准.md`](../../agents/standard/计划书标准.md) |
| 任务链 | 围绕同一目标连续推进的 `execute / review / merge` 过程，默认沿用同一记录文件 | 单次会话消息，或互不相关的两个任务 | [`agents目录规则.md`](../../agents/standard/agents目录规则.md) |
| `worktree` | 当前任务独立工作目录；常规任务日志和任务范围内改动默认都落在这里 | 主仓根目录，或任意临时文件夹 | [`agents目录规则.md`](../../agents/standard/agents目录规则.md) |
| 主仓 | 仓库根目录的主工作区，用于维护 `TODO.md`、`DONE.md`、角色名单和共享标准文档基线 | 某个具体任务的独立 `worktree` | [`agents目录规则.md`](../../agents/standard/agents目录规则.md) |
| 阻塞记录 | 因 `worktree` 缺失、文件范围不清、资产缺失等原因无法继续推进时，必须先写入记录文件的日志 | 审查结论，或普通完成记录 | [`异常处理规范.md`](../../agents/standard/异常处理规范.md) |
| 待确认记录 | 当前还不能继续，但问题主要是语义、权限、范围或脚本口径待澄清时写入的记录 | 已确认通过的实现说明，或最终结论 | [`异常处理规范.md`](../../agents/standard/异常处理规范.md) |
| 归档任务 | 为计划书归档单独创建的任务；只有完成归档任务后，管理员才能执行 `-done-plan` | 直接口头通知归档，或普通 `merge` 任务 | [`协作执行通用规则.md`](../../agents/standard/协作执行通用规则.md) |
| 公开 `spec / API` | 计划书或 `spec` 正文中明确对外固定的接口、参数顺序、样例与边界 | 实现中的临时 helper，或只在任务记录里提过的口头口径 | [`计划书标准.md`](../../agents/standard/计划书标准.md) |
| 验收资产 | 用于锁定最终效果的 `expectation`、`pytest`、脚本命令或执行者需补建的校验入口 | 任务记录本身，或无法执行的抽象描述 | [`计划书标准.md`](../../agents/standard/计划书标准.md) |

## 使用提醒

- 若一个术语在其他文档里出现歧义，优先以本页定义为准，再回看对应标准文档的上下文。
- 本页只统一高频治理术语，不替代各文档对具体流程、权限和模板的正文约束。
