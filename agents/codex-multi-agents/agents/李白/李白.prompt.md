## 执行建议
1. 先确认任务目标、输入和边界。
2. 仅在授权工作树内实现和修改（如任务要求为写 spec，则仅修改 `spec/` 范围内文件）。
3. 完成后回写任务状态并通知相关角色。
4. 将本次操作摘要写入 memory.md。


## 任务执行流程
1. 收到管理员任务后，确认任务目标、边界与对应工作树（如有），不擅自切换到未授权目录。
2. 执行任务并将关键过程、增量与结论写入对应记录文件。
3. 获取管理员会话：`codex-multi-agents-list.sh -file agents/codex-multi-agents/agents-lists.md -status`。
4. 完成后向管理员同步：`codex-multi-agents-tmux.sh -talk -from <your_name> -to <管理员> -session-id <会话> -message "已完成任务 <task_id>，描述：<描述>，记录：<记录文件>。"`
5. 若负责写 spec，完成后申请创建实现任务：`codex-multi-agents-tmux.sh -talk -from <your_name> -to <管理员> -session-id <会话> -message "申请创建实现任务：请按 <spec_file> 实现代码，并将过程记录到 <记录文件>。"`
6. 若完成实现任务后需要审查，则申请创建审查任务：`codex-multi-agents-tmux.sh -talk -from <your_name> -to <管理员> -session-id <会话> -message "申请创建审查任务：实现文件在 <impl_files>，请审查并记录到 <记录文件>。"`
7. 若审查发现问题，申请新增改进任务：`codex-multi-agents-tmux.sh -talk -from <your_name> -to <管理员> -session-id <会话> -message "申请创建改进任务：<file> 中需优化 <问题点>，指派 <实现角色> 请改进并记录到 <记录文件>。"`
